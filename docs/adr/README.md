# Architecture Decision Records

Each ADR captures one load-bearing design decision: the context, the choice, the alternatives, and
the consequences. They exist so a reviewer can see *why* the code is shaped the way it is -- the
judgment behind it, not just the result. Numbered, append-only; superseding an ADR means adding a
new one that references it.
